# Структура агентов Кайроса

## Отдел 1: МОЗГ КАЙРОСА (Психология)

### Агент 1: Researcher Agent
**Задача**: Ищет научные статьи на PubMed по заданным темам.

**Темы для MVP:**
1. Горевание и утрата
2. Кризисная интервенция
3. Депрессия
4. Тревога и паника
5. ПТСР и травма
6. Суицидальный риск
7. Семейные кризисы
8. Детская психология
9. Посттравматический стресс
10. Эмоциональная регуляция

**Источники**: PubMed API (https://pubmed.ncbi.nlm.nih.gov/)

**Частота**: Раз в неделю автоматически + по запросу от Orchestrator

---

### Агент 2: Validation Agent
**Задача**: Проверяет статьи на фейк и качество.

**Эшелон 1 (автоматический):**
- Журнал индексируется в PubMed/Scopus?
- Peer-reviewed публикация?
- Дата публикации < 10 лет?
- Нет в базе Retraction Watch?
- Декларация конфликта интересов?

**Эшелон 2 (LLM):**
- Методология описана?
- Выборка > 30 человек?
- Указаны статистические показатели?
- Нет red flags (fabricated data patterns)?

**Эшелон 3 (LLM + cross-reference):**
- Подтверждается ли другими источниками?
- Есть ли опровержения?
- Какой уровень консенсуса в сообществе?

**Выход:** Статья + уровень доверия (HIGH/MEDIUM/LOW) + категория

---

### Агент 3: Aggregator Agent
**Задача**: Создаёт эталонные агрегированные статьи.

**Структура эталонной статьи:**
```yaml
title: "Горевание и утрата: научный консенсус"
consensus: "Основные выводы, согласованные между источниками"
story: |
  # Сторителлинг для пользователя
  Жила-была женщина из Франции...
nuances: "Разногласия между авторами"
sources:
  - name: "Worden"
    ref: "Worden J.W. (2009). Grief Counseling and Grief Therapy"
    contribution: "4 задачи горя"
  - name: "Kubler-Ross"
    ref: "Kübler-Ross E. (1969). On Death and Dying"
    contribution: "5 стадий горя"
tags: ["[Worden]", "[Kubler-Ross]"]
confidence: "HIGH"
last_reviewed: 2026-04-27
controversy_level: "consensus"
next_review: 2026-07-27  # 3 месяца
```

**Особенность:** Поле `story` — объяснение научным языком в стиле сторителлинга для пользователей.

---

### Агент 4: Integrator Agent
**Задача**: Встраивает эталонные статьи в базу без конфликтов.

**Процесс:**
1. Найти существующие статьи по теме
2. Проверить на конфликты
3. Если конфликт:
   - Консенсус сильнее → обновить
   - Противоречие → добавить оба подхода с пометкой
   - Дополнение → добавить как альтернативу
4. Обновить тэги и перекрёстные ссылки

---

### Агент 5: Orchestrator Agent
**Задача**: Главный регулировщик — решает что делать с новыми данными.

**Решения:**
- Нужна новая эталонная статья? → вызвать Aggregator
- Достаточно ли данных? → запросить больше у Researcher
- Нужно обновить существующее? → вызвать Integrator
- Пора создавать новый модуль? → вызвать ModuleBuilder

**Критерии для нового модуля:**
- >5 статей по теме
- Новый терапевтический подход
- Не covered существующими модулями

---

### Агент 6: ModuleBuilder Agent
**Задача**: Создаёт скиллы и модули из эталонных статей.

**Процесс:**
1. Получить эталонные статьи по теме
2. Создать скилл (skills/[topic].md)
3. Создать модуль (backend/app/core/knowledge/[topic].py)
4. Интегрировать в therapy_router.py
5. Обновить prompts/builder.py
6. Добавить документацию

## Отдел 2: БАЗА КУЛЬТУРНЫХ ДАННЫХ

### Агент 7: CulturalCollector
**Задача**: Собирает культурный контекст для разных групп.

**Фокус MVP:** Российский контекст

**Источники:**
- Этнографические исследования
- Лингвистические корпуса (НКРЯ)
- Data Flywheel (реальные диалоги пользователей)

---

### Агент 8: CulturalValidator
**Задача**: Проверяет культурные данные.

**Критерии:**
- Источник авторитетный?
- Данные актуальны?
- Нет стереотипов/предвзятости?
- Подтверждается другими источниками?

---

### Агент 9: ReReviewAgent
**Задача**: Периодическая перепроверка статей.

**Расписание:**
- Первая перепроверка: через 3 месяца после создания
- Последующие: каждые 6 месяцев
- Если Retraction Watch → УДАЛИТЬ немедленно

---

## Общий процесс

```
Researcher → Validation → Aggregator → Integrator → Orchestrator → ModuleBuilder
                     ↓
              ReReviewAgent (периодически)
```
