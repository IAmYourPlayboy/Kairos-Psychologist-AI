# Система памяти о пользователе (User Memory System)

> **Концепция**: "Досье пользователя" — структурированная база знаний, которая автоматически пополняется во время диалога и доступна в любом чате.
> **Вдохновлено**: Moonshot AI memory system
> **Версия**: 1.0 | **Дата**: Апрель 2026

---

## Проблема

**Сейчас**: Каждый чат — это новый "Кайрос", который ничего не знает о пользователе.

```
Чат 1: "Меня зовут Иван, мне 30 лет, работаю программистом, недавно развёлся"
Чат 2: "Привет!" → Кайрос: "Привет! Чем могу помочь?" (не помнит про Ивана)
```

**Проблемы**:
- Пользователь вынужден повторять информацию о себе в каждом чате
- Нет преемственности между сессиями
- Контекстное окно забивается повторяющейся информацией
- Невозможно отслеживать прогресс пользователя между сессиями

---

## Решение: Досье пользователя

**Концепция**: Структурированная база знаний о пользователе, которая:
1. **Автоматически пополняется** — LLM извлекает факты из диалога и сохраняет в досье
2. **Структурирована по категориям** — детство, работа, отношения, здоровье, цели, etc.
3. **Доступна в любом чате** — загружается в начале каждой сессии
4. **Компактна** — занимает ~500-1000 токенов вместо 10K+ токенов контекста
5. **Обновляется динамически** — новые факты добавляются, устаревшие удаляются

### 2. Как это работает

#### Шаг 1: Извлечение фактов из диалога

```python
# После каждого сообщения пользователя
user_message = "Меня зовут Иван, мне 30 лет, недавно развёлся с женой"

# LLM извлекает факты
facts = extract_facts(user_message)
# [
#   {"category": "basic_info", "field": "name", "value": "Иван"},
#   {"category": "basic_info", "field": "age", "value": 30},
#   {"category": "relationships", "field": "current_status", "value": "разведён"},
#   {"category": "current_challenges", "value": {"challenge": "развод", "severity": "high"}}
# ]

# Обновить досье
update_user_dossier(user_id, facts)
```

#### Шаг 2: Загрузка досье в начале сессии

```python
# При создании нового чата
dossier = load_user_dossier(user_id)

# Сжать досье в компактный промпт (~500 токенов)
dossier_prompt = compress_dossier(dossier)

# Добавить в системный промпт
system_prompt = f"""
{BASE_PROMPT}

## ИНФОРМАЦИЯ О ПОЛЬЗОВАТЕЛЕ (из досье)

{dossier_prompt}

Используй эту информацию для персонализации ответов.
НЕ спрашивай повторно то, что уже знаешь.
Если информация устарела — обнови досье.
"""
```

#### Шаг 3: Обновление досье во время диалога

```python
# После каждого обмена сообщениями
if new_facts_detected:
    update_user_dossier(user_id, new_facts)
    
# Если факт устарел
if fact_outdated:
    remove_from_dossier(user_id, fact_id)
```

## Преимущества

### 1. Преемственность между сессиями

```
Чат 1 (01.04.2026):
User: "Меня зовут Иван, недавно развёлся"
Kайрос: "Понимаю, развод — это тяжело. Расскажи подробнее?"
[Досье обновлено: name=Иван, current_challenge=развод]

Чат 2 (15.04.2026):
User: "Привет, мне плохо"
Кайрос: "Привет, Иван. Слышу, что тебе тяжело. Это связано с разводом?"
[Кайрос помнит про Ивана и развод из досье]
```

### 2. Персонализация подхода

```python
# Кайрос знает, что работает для этого пользователя
if user_dossier["coping_strategies"]["what_works"]:
    # Предложить то, что уже помогало
    suggest_technique("прогулки на природе")

if user_dossier["therapy_history"]:
    # Не предлагать то, что не сработало
    avoid_technique("SFBT шкалирование")
```

### 3. Отслеживание прогресса

```python
# Сравнить текущее состояние с прошлым
previous_distress = user_dossier["therapy_history"][-1]["distress_before"]
current_distress = 0.5

if current_distress < previous_distress:
    # "Вижу, что тебе стало легче с прошлого раза"
```

### 4. Экономия контекста

**Без досье**: 10 000+ токенов контекста для хранения истории
**С досье**: ~500-1000 токенов компактного промпта

### Файлы проекта

```
backend/app/core/user_memory/
├── README.md ................... Этот файл
├── dossier.py .................. Модель досье (Pydantic)
├── extractor.py ................ Извлечение фактов из диалога (LLM)

├── storage.py .................. Сохранение/загрузка из PostgreSQL
└── updater.py .................. Обновление досье
```

### База данных

```sql
CREATE TABLE user_dossiers (
    user_id UUID PRIMARY KEY,
    dossier JSONB NOT NULL,
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW(),
    version INTEGER DEFAULT 1
);

-- Индекс для быстрого поиска
CREATE INDEX idx_user_dossiers_user_id ON user_dossiers(user_id);
```

### API

```python
# Загрузить досье
GET /api/user/dossier

# Обновить досье (автоматически после каждого сообщения)
POST /api/user/dossier/update
{
    "facts": [
        {"category": "basic_info", "field": "name", "value": "Иван"}
    ]
}

# Удалить факт из досье
DELETE /api/user/dossier/fact/{fact_id}
```

## Приватность и безопасность

### 1. Анонимизация

Досье хранится **анонимно**:
- `user_id` = UUID (не связан с email/телефоном)
- Имена заменяются на псевдонимы в data flywheel
- ПДн (адреса, телефоны) не сохраняются

### 2. Контроль пользователя

Пользователь может:
- **Просмотреть досье** — увидеть, что Кайрос о нём знает
- **Удалить факты** — убрать неверную или чувствительную информацию
- **Очистить досье** — удалить всё

### 3. Согласие

При первом использовании:
```
Кайрос может запоминать информацию о тебе между сессиями,
чтобы не спрашивать повторно. Это помогает персонализировать помощь.

Вся информация хранится анонимно и доступна только тебе.

[Разрешить] [Запретить] [Подробнее]
```

## Сравнение с контекстным окном

| Параметр | Контекстное окно | Досье пользователя |
|----------|------------------|-------------------|
| **Размер** | 10 000+ токенов | 500-1000 токенов |
| **Доступность** | Только в текущем чате | Во всех чатах |
| **Структура** | Неструктурированный текст | Структурированный JSON |
| **Обновление** | Автоматически забывается | Сохраняется навсегда |
| **Поиск** | Сложно найти нужное | Быстрый доступ по категориям |
