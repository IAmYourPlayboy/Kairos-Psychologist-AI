## Зачем

Старый rule-based детектор (`crisis/detector.py`) и селектор ветки
(`branch_selector.py`) удаляются в Фазе 6, потому что они не понимают
намёков и контекста. Им на смену — **слой восприятия** из четырёх
связанных компонентов:

1. **Brain** (статичные знания, в `core/knowledge/`) — терапевтические
   протоколы, эталонные статьи. Меняется редко, через цепочку агентов.
2. **Dossier** (на user_id) — папки → подпапки → факты с цитатами.
   Обновляется фоновым ReflectionAgent через 15 минут после последнего
   сообщения пользователя.
3. **Mood** (на session_id, в Redis) — 6 осей внутреннего состояния
   Кайроса (alertness, warmth, pace, assertiveness, trust_in_user, depth).
   Обновляется правилами после каждого сообщения.
4. **MessageAnalyzer** — отдельный LLM-вызов на каждое сообщение.
   Возвращает структурированный JSON: risk_level, эмоции, тема, скрытые
   сигналы, что нужно пользователю, какие папки досье подтянуть,
   inner_monologue («мысли» Кайроса для админки).

См. дизайн-документ для деталей.
