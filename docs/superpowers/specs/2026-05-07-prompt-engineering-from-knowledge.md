# Promпт-инжиниринг: выжимки из knowledge/ в три LLM-промпта

> **Создан**: Сессия 23, 7 мая 2026
> **Связь**: Фаза 1.3 в [`PROGRESS.md`](../../../PROGRESS.md), является Уровнем 1 в [`mood-calibration-strategy.md`](2026-05-07-mood-calibration-strategy.md).
> **Статус**: design doc для предстоящей работы.

---

## Проблема

Сейчас 75 КБ ценного материала в `backend/app/core/knowledge/` (SIX C's Фарчи, ВОЗ PFA, CBT, DBT, ACT, SFBT, MI) подключены **только к одному месту** — `prompts/base.py`, и то — через **упоминание**, не содержание.

Конкретно:
- `prompts/base.py` строка 25: `«Ты работаешь по доказательным протоколам (SIX C's Фарчи, ВОЗ PFA).»` — упомянуто, но **сами протоколы LLM не видит**.
- `MessageAnalyzer` (`analyzer_prompt.py`): для оценки `risk_level` имеет короткие 4 строки описания уровней — без поведенческих маркеров из ВОЗ PFA «assess_distress».
- `ReflectionAgent extract` (`reflection_prompt.py`): извлекает «факты», но не имеет критериев «что считать **триггером**» из CBT (когнитивные искажения) и DBT (эмоциональная дисрегуляция).
- Основной `build_main_prompt` в `core/perception/prompt_builder.py`: **вообще** не подключает SIX C's и WHO PFA. Сессия 18 убрала старый `branch_a/branch_b` подход (который их подключал по веткам), а замены не появилось.

Результат: LLM «знает» термины протоколов, но не имеет конкретных техник перед глазами — действует по своему усмотрению.

---

## Архитектура решения

### Принцип «один промпт — одна роль»

Три LLM-вызова с разными задачами требуют **разных** выжимок:

| Промпт | Задача | Какая выжимка нужна |
|---|---|---|
| `analyzer_prompt.py` | Решить `risk_level`, `theme`, `folder_hints` | **Поведенческие маркеры** уровней дистресса по WHO PFA «LOOK» |
| `reflection_prompt.py` (extract) | Извлечь устойчивые факты | **Критерии триггера** по CBT/DBT (как отличить от разовой реакции) |
| `prompt_builder.py` (основной) | Сгенерировать ответ пользователю | **Конкретные техники** SIX C's / WHO PFA / CBT / DBT / ACT, выбранные по `theme` и `risk_level` |

### Единый источник правды: `app/core/knowledge/digests.py`

Все выжимки хранятся в **одном модуле**, импортируются в три промпта. Это:
- Предотвращает расхождение (изменение в одном месте = изменение для всех)
- Облегчает ревизию (все выжимки в одном файле)
- Повторяет паттерн `folders.py` (один источник для analyzer/reflection)

### Подбор техник по ситуации (только в основном промпте)

В `build_main_prompt` добавляется блок «Подходящие техники». Содержимое выбирается функцией `_pick_technique_digest(report)`:

```python
def _pick_technique_digest(report: PerceptionReport) -> str | None:
    """Подобрать выжимку техник под ситуацию.

    Логика:
    - immediate → None (CRISIS_PROMPTS уже жёстко предписывает поведение)
    - high → PFA_VALIDATION_DIGEST (валидация перед предложением контактов)
    - elevated и эмоция страх/паника → PFA_GROUNDING_DIGEST (5-4-3-2-1, дыхание)
    - theme содержит "семья/конфликт/безысходность" → SIX_CS_DIGEST
    - theme содержит "утрата" → ACT_GRIEF_DIGEST
    - theme содержит "тревога" без кризиса → CBT_ANXIETY_DIGEST
    - эмоция гнев → DBT_ANGER_DIGEST
    - иначе → None
    """
```

Если ничего не подошло — техника не подмешивается, основная LLM использует только базовый промпт + `CRISIS_PROMPTS` + Mood + Dossier (как сейчас).

## Размеры выжимок

Каждая выжимка — **5-30 строк**. Принципы:

- **Концентрация на действиях**: «вот что делать», не «вот история подхода».
- **Примеры конкретных фраз**: «Поставь ноги на пол», «Найди жёлтый предмет» — модель копирует стиль.
- **Запреты явно**: «НЕ говори X в этой ситуации».
- **Без академического языка**: бот должен быть человеком, не учебником.

Это разительно отличается от полного `who_pfa.py` (331 строка с dataclass'ами и метаданными). dataclass'ы хороши для кода, но в промпте они — лишний шум.

## ADR (architectural decision records)

### ADR-1: один файл `digests.py`, не разбросанные строки

**Решение**: Все выжимки централизованы в `app/core/knowledge/digests.py`.

**Альтернатива отвергнута**: «Положить выжимку для analyzer в `analyzer_prompt.py`, для reflection — в `reflection_prompt.py`, для main — в `prompt_builder.py`».

**Почему отвергнуто**: дублирование при пересечениях (например, ВОЗ PFA-валидация нужна и analyzer, и main), плюс сложнее ревизовать что именно LLM «знает».

### ADR-2: выжимки в коде, не в БД

**Решение**: Тексты выжимок — Python константы в `digests.py`.

**Альтернатива отвергнута**: «Хранить выжимки в БД-таблице `prompt_digests` чтобы их можно было редактировать на лету».

**Почему отвергнуто**: для изменения промпта (тонкая работа с поведением LLM) нужен code review, тестирование, git история. Редактирование на лету через БД — это рецепт случайных регрессий поведения бота. Если БД-конфигуратор появится в Фазе 9+ — он будет предложением правок, не автодеплоем.

### ADR-3: подбор техник только в основном промпте, не в analyzer

**Решение**: `_pick_technique_digest()` вызывается **только** из `build_main_prompt`. Analyzer всегда видит **полный** WHO_PFA_DISTRESS_LEVELS независимо от темы.

**Альтернатива отвергнута**: «Тоже подмешивать релевантную тему в analyzer чтобы он был "глубже"».

**Почему отвергнуто**: analyzer — это **классификатор**, не терапевт. Его задача — поставить правильный risk_level, для этого нужны общие маркеры из WHO PFA, не специфические техники CBT/DBT. Подмешивание техник может его «толкать» к подсказке темы — а это уже искажение.

### ADR-4: при `risk_level=immediate` техники НЕ подмешиваются

**Решение**: для `immediate` `_pick_technique_digest` возвращает None. Основная LLM получает только `CRISIS_PROMPTS["immediate"]` + базу.

**Почему**: `CRISIS_PROMPTS["immediate"]` уже содержит **жёсткое** предписание («НЕ задавай вопросов, СРАЗУ давай контакты»). Подмешивание SIX C's или WHO PFA техник может привести к тому, что бот начнёт сначала «грунтить» вместо немедленных контактов — это опасный регресс.

### ADR-5: смена выжимки не должна ломать существующие 304 теста

**Решение**: Все правки промптов — это **расширение** контекста, не изменение интерфейса. `PerceptionReport` не меняется. `build_main_prompt` принимает те же аргументы. API не меняется.

**Тест**: после всех правок прогон `pytest -v` должен дать 304/304.

**Что добавляется** (но не ломает существующее):
- 1-2 smoke-теста: «при theme=паника основной промпт содержит фразу из PFA_GROUNDING_DIGEST», «analyzer-промпт содержит маркеры immediate-уровня».

## Структура файла `digests.py`

```python
"""
Компактные выжимки из knowledge/*.py для подмешивания в LLM-промпты.

Используются:
- analyzer_prompt.py — WHO_PFA_DISTRESS_LEVELS
- reflection_prompt.py — CBT_TRIGGERS, DBT_TRIGGERS
- perception/prompt_builder.py — SIX_CS_DIGEST, PFA_*, CBT_*, DBT_*, ACT_*

Принципы (см. spec 2026-05-07-prompt-engineering-from-knowledge.md):
- 5-30 строк на выжимку
- Конкретные фразы и действия, не теория
- Явные запреты («НЕ говори X»)
- Без академического языка

# === Для MessageAnalyzer ===

```
