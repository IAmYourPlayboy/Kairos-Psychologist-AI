> **Дата**: 2026-05-12 (Сессия 29)
> **Дизайн**: [`2026-05-12-code-audit-systemic-cleanup-design.md`](../specs/2026-05-12-code-audit-systemic-cleanup-design.md)

> **Место в роадмапе**: между Фазой 1.5 (UserState) и Фазой 2 (PubMed-агенты). Подфазы 1.6.1-1.6.7 делаем до или параллельно с 1.5. Подфазы 1.6.8-1.6.9 — после 1.5.1.

---

## Общие правила выполнения

1. **Каждая подфаза — отдельный коммит**, не объединять. Ревью diff'а проще когда он сфокусирован.
2. **Между подфазами — `pytest -v` должен быть зелёный** (или с заранее ожидаемым списком удалённых тестов, зафиксированным в commit message).
3. **Никакого нового поведения наружу** в этой фазе. API контракты, DB схема (кроме ORM-сплита), промпты в LLM — не меняются. Это *внутренний* cleanup.
4. **Ничего из `backend/agents/` и `backend/tests/` не трогаем** — согласовано в дизайне.
5. **Все изменения — в одной ветке** (например `phase-1.6-cleanup`), мержим в main только после прохождения всей фазы.
6. **Frontend в этой фазе практически не трогаем** — только 4.10 (`AgeGroup` в `types.ts`). Всё остальное — backend.

## 1.6.3 `TechniqueDigest` схема (Pydantic)

**Цель**: единая строгая схема для всех терапевтических техник, замена 6 разных dataclass'ов.

### Задачи

1. **Создать `backend/app/core/knowledge/schema.py`**:

   ```python
   """Единая схема для терапевтических техник (Фаза 1.6.3, Сессия 29).

   См. дизайн: docs/superpowers/specs/2026-05-12-code-audit-systemic-cleanup-design.md (4.2).
   """

   from __future__ import annotations
   from typing import Literal
   from pydantic import BaseModel, Field


   TrustLevel = Literal["core", "validated", "experimental"]
   AgeGroupValue = Literal["child", "youth", "adult"]


   class ForbiddenPhrase(BaseModel):
       """Одна ситуативная запрещённая фраза для этой техники."""
       phrase: str = Field(..., description="Сама фраза, которую НЕ говорить")
       reason: str = Field(..., description="Почему нельзя именно в этом контексте")
       alternative: str | None = Field(None, description="Что сказать вместо, если есть")


   class SelectionRule(BaseModel):
       """Правило при каком состоянии пользователя выбирается этот digest."""
       risk_levels: list[Literal["normal", "elevated", "high", "immediate"]]
       emotion_markers: list[str] = Field(default_factory=list)
       theme_markers: list[str] = Field(default_factory=list)
       age_groups: list[AgeGroupValue] = Field(default_factory=lambda: ["adult"])
       priority: int = Field(..., description="При совпадении нескольких — выигрывает больший")
       exclude_if_risk: list[Literal["normal", "elevated", "high", "immediate"]] = Field(
           default_factory=list,
           description="Не применять при этих уровнях риска (для immediate — ничего не подмешиваем)",
       )


   class TechniqueDigest(BaseModel):
       """Единая модель терапевтической техники."""
       id: str = Field(..., description="Machine-readable ID, уникален в ALL_DIGESTS")
       name: str = Field(..., description="Человеко-читаемое название")
       approach_label: str = Field(..., description='"ВОЗ PFA — ЗАЗЕМЛЕНИЕ", "SIX C\'s", используется в заголовке блока промпта')
       category: Literal["PFA", "CBT", "DBT", "ACT", "SFBT", "SIX_CS"]
       when_applies: str = Field(..., description="Описание ситуации для разработчика, НЕ идёт в промпт")
       body_for_prompt: str = Field(..., description="Готовый markdown-текст блока, идёт в LLM как есть")
       forbidden_phrases_in_context: list[ForbiddenPhrase] = Field(default_factory=list)
       selection_rule: SelectionRule
       source: str = Field(..., description="Научный источник: PubMed DOI, или эталонная статья, или 'manual (Сессия NN)'")
       trust_level: TrustLevel = "core"
   ```

   минус «ЧЕГО ИЗБЕГАТЬ» блок — он теперь в forbidden_phrases_in_context)""",
       forbidden_phrases_in_context=[
           ForbiddenPhrase(

           ForbiddenPhrase(
               phrase="время лечит",
               reason="обесценивание — иногда боль остаётся надолго",

               phrase="бывает и хуже",
               reason="минимизация страданий пользователя",

## Если что-то ломается

- **pytest упал после подфазы** — откат в пределах подфазы (one commit = one rollback), разбираемся.
- **Приложение не импортируется** — `git bisect` на коммитах подфазы.
- **Удалили нужное** — восстанавливаем из git (`git show` + cherry-pick частично). `from_dead_code.md` идеи всё равно остались.
- **Ruff UP изменил больше чем ожидали** — просматриваем diff вручную, непонятные правки откатываем через `git restore -p`.
