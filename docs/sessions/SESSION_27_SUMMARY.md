## Архитектурные решения Сессии 27 (зафиксированы в CLAUDE.md)

### Принцип «без хардкодных фраз»

Кайрос НЕ использует готовых шаблонных фраз в ответе. LLM формулирует каждый ответ сама и заново.

В промптах **нет** строк вроде «Я слышу тебя. Прямо сейчас позвони: ...», «Тебе сейчас тяжело. Я правильно понимаю?» как примеров фраз бота. Всё такое — примеры для разработчика-читателя, но не инжектится в промпт LLM.

В промптах **остаётся**:
1. **Протокол** — описание поведения («при immediate: минимум слов до контактов»)
2. **Запреты** — список фраз которых НЕЛЬЗЯ говорить (защитная граница)
3. **Данные** — конкретные факты (телефоны, выжимки из knowledge/)
4. **Формат вывода** — технические правила

Зачем: (а) уникальность разговора — пользователь чувствует что **этот** ответ ему; (б) защита от эффекта Character.AI; (в) простота поддержки, меньше токенов.

**Применение**: `backend/app/core/prompts/crisis.py` переписан по этому принципу в Сессии 27.

### Двухуровневая система запретов

Один глобальный список запретов на все ситуации — тупик: «Бывает и хуже» в кризисе суицида ужасно, в разговоре про сломанный ноготь — нормально. Плюс разрастётся до тысячи фраз.

Поэтому:

**Уровень 1 — глобальное ядро (~15-20 запретов)**: то что Кайрос не говорит НИКОГДА. `prompts/base.py`, в каждом промпте. («Я понимаю что ты чувствуешь», «Всё будет хорошо», «Держись» и т.д.)

**Уровень 2 — ситуативные запреты (по 3-10 на тему)**: живут в `core/knowledge/*.py`, подмешиваются когда analyzer определил тему. Примеры: при утрате не говорить «время лечит», при родителе-алкоголике у ребёнка не говорить «поговори с родителями».

**PubMed-агенты (Фаза 2)** будут извлекать `forbidden_phrases_in_context` как обязательное поле эталонной статьи — база запретов будет расти автоматически вместе с базой знаний.

## Child Mode — принятая идея (Фаза 3.7 в PROGRESS.md)

Идея пользователя: Кайрос как фигура, частично замещающая родителей для детей из токсичных семей.

**Переформулировка**: не «замена родителей» (опасный фрейм, parasocial attachment, пример Character.AI), а **«временный достаточно хороший собеседник + мостик к живому взрослому»** (Винникоттовский transitional object). После N сессий без упоминания живого взрослого — бот мягко возвращает к мысли «рядом должен быть живой человек».

**Обоснование**: иск Garcia v. Character.AI (октябрь 2024, 14-летний Sewell Setzer III, суицид). Центральный аргумент — «продукт сознательно поощрял эмоциональную зависимость у подростка».

**6 ADR** зафиксированы в PROGRESS.md Фаза 3.7:
1. Child Mode — надстройка, не отдельный продукт
2. Переключение — явное, не автоматическое (через onboarding/invite)
3. Обязательная механика мостика к живому взрослому
4. Доступ только через партнёрский канал (кризисный центр / школьный психолог)
5. Никакой персонализации под «замену конкретного человека» (голоса/имена запрещены)
6. Отдельная обкатка 30 детских сценариев с консультацией детского психолога

**10 подфаз**: 3.7.1 промпт → 3.7.2 кризисные промпты → 3.7.3 child_digests → 3.7.4 мостик → 3.7.5 invite → 3.7.6 30 сценариев → 3.7.7 научная база → 3.7.8 юр.рамка → 3.7.9 UI → 3.7.10 тесты.

**⚠️ Порядок выполнения переставлен в Сессии 27**: СНАЧАЛА 3.7.7 (научная база из PubMed), ПОТОМ 3.7.3 (digest), ПОТОМ 3.7.1 (промпт). Причина: нельзя писать детские промпты из головы разработчика, детская психология — научная область со своим консенсусом (Боулби, ACE, ISTSS).

**Фундамент Child Mode уже заложен в Фазе 0.5**: `age_group` протянут через весь pipeline.

## Ребрендинг «ППП-бот» → «Кайрос»

Старые документы писались до придумывания имени «Кайрос». Ребрендинг произведён только в живых документах (CLAUDE.md, PROGRESS.md, journal.md, scenarios.md, skills/*), исторические specs/plans/sessions из прошлых сессий **не тронуты** как архив решений.

**Принятая терминология**:
- **Кайрос** — имя продукта (везде где раньше «ППП-бот»)
- **ППП** или **ППП-режим** — название одного из двух режимов работы Кайроса (базовый чат кризисной помощи). Противопоставляется **режиму цифрового двойника**.
- **ППП** как аббревиатура (Первая Психологическая Помощь) — методологический термин, не меняется.
