## Концепция

Каждая модель использует **свой токенизатор**, поэтому количество токенов в одном и том же тексте будет отличаться. При работе через OpenAI-совместимое API количество использованных токенов возвращается в каждом ответе модели в поле `usage`.

Модели Яндекса используют токенизатор, **специально оптимизированный для русского языка**. Это позволяет увеличить среднее количество символов в токене и уменьшить стоимость обработки текста.

Токенизатор бесплатный (вызовы TokenizerService и методы Tokenizer не тарифицируются). Для использования нужна роль **`ai.languageModels.user`** или выше на каталог.

## Сравнение эффективности токенизаторов

⚠️ **Очень важная таблица для нашего проекта**: показывает что **YandexGPT Pro в среднем тратит на 30-40% МЕНЬШЕ токенов на тот же русский текст**, чем Qwen3 235B.

### Русский текст (501 символ)

| Модель | Токенов | Среднее символов/токен |
|---|---|---|
| YandexGPT Pro | **96** | **5,2** |
| Qwen3 235B | 139 | 3,6 |
| gpt-oss-120b | 109 | 4,6 |

> На том же русском тексте Qwen3 тратит **на 45% больше токенов** (139 vs 96), чем YandexGPT Pro.

### Английский текст (477 символов)

| Модель | Токенов | Среднее символов/токен |
|---|---|---|
| Alice AI LLM | 89 | 5,36 |
| Qwen3 235B | 87 | 5,48 |
| gpt-oss-120b | 87 | 5,48 |

> На английском разница незначительная — все три модели близки. Разница ярко проявляется именно на русском.

## Что это значит для наших расчётов

При сравнении цен Qwen vs YandexGPT нужно учитывать токенизатор:

**Например**, типичное сообщение пользователя ≈ 200 символов на русском:
- YandexGPT Pro: 200 / 5,2 ≈ **38 токенов**
- Qwen3 235B: 200 / 3,6 ≈ **56 токенов**

То есть Qwen на русском тексте обработает на **~47% больше токенов**. Это нужно умножить на разницу в цене за 1К токенов.

**Если Qwen3.6 35B стоит 0,2 ₽ за 1К входных** vs **YandexGPT Lite 0,2 ₽ за 1К входных**:
- За тот же 200-символьный текст:
  - YandexGPT Lite: 38 токенов × 0,2 ₽ / 1000 = **0,0076 ₽**
  - Qwen3.6 35B: 56 токенов × 0,2 ₽ / 1000 = **0,0112 ₽**
- Разница ~47% не в пользу Qwen.

⚠️ Это контр-аргумент к выводу «Qwen3.6 35B и YandexGPT Lite одинаково стоят» — на русском Qwen фактически дороже.

## API токенизатора

### Метод `Tokenize` (REST)

```
POST https://llm.api.cloud.yandex.net/foundationModels/v1/tokenize
```

Тело запроса:
```json
{
  "modelUri": "string",
  "text": "string"
}
```

- `modelUri` — идентификатор модели для токенизации
- `text` — текст для токенизации

Ответ:
```json
{
  "tokens": [
    {
      "id": "string",
      "text": "string",
      "special": "boolean"
    }
  ],
  "modelVersion": "string"
}
```

Где `tokens[]`:
- `id` — внутренний идентификатор токена (int64)
- `text` — текстовое представление токена
- `special` — является ли токен специальным (управляет поведением модели, не виден пользователям)

## Пример (для YandexGPT Pro)

Запрос:
```bash
export IAM_TOKEN=<IAM-токен>
curl --request POST \
  --header "Authorization: Bearer ${IAM_TOKEN}" \
  --data '{
    "modelUri": "gpt://<идентификатор_каталога>/yandexgpt",
    "text": "Управление генеративными моделями осуществляется с помощью промптов..."
  }' \
  "https://ai.api.cloud.yandex.net/foundationModels/v1/tokenize"
```

Ответ возвращает массив токенов вида:
```json
{
  "tokens": [
    { "id": "1", "text": "<s>", "special": true },
    { "id": "19078", "text": "▁Управление", "special": false },
    { "id": "10810", "text": "▁генера", "special": false },
    { "id": "26991", "text": "тивными", "special": false },
    ...
  ],
  "modelVersion": "09.02.2025"
}
```

Видно, что токенизатор Яндекса **режет слова на семантические куски** (например, «генеративными» → «генера» + «тивными»), а не на BPE-куски символов. Это и даёт высокую эффективность на русском.
