## TL;DR

Qwen 3.6 заменяет Qwen 3.5. Цена та же. Главное — обновить `model_uri`.

```diff
- gpt://<folder>/qwen3.5-35b-a3b-fp8
+ gpt://<folder>/qwen3.6-35b-a3b
```

> ⚠️ До **28 мая 2026** обращения к старому URI автоматически перенаправляются на Qwen 3.6.
> После 28 мая старый URI перестанет работать.

## Что улучшилось в Qwen 3.6 vs 3.5

> Qwen 3.6 заменит прошлую версию мультимодальной языковой модели — Qwen 3.5.
> Новая версия значительно превосходит своего прямого предшественника, особенно в задачах **агентного кодинга и логического вывода**.

1. **Улучшения в рассуждениях** — заметный прирост в бенчмарках, связанных с рассуждениями, особенно в:
   - математике
   - логике
   - решении многошаговых задач

2. **Более точное следование инструкциям** — модель стала лучше:
   - придерживаться структурированных форматов вывода
   - соблюдать ограничения
   - точнее обрабатывать нюансы инструкций без смещения

3. **Повышение качества генерации кода** — прирост качества заметен в задачах с:
   - объёмной кодовой базой
   - менее распространёнными языками программирования

> ⚠️ **Заметка для Кайроса**: улучшения 1 и 2 (особенно «следование инструкциям» и «структурированные форматы вывода») — это ровно то, что нам нужно для `MessageAnalyzer`. Это +1 в пользу Qwen 3.6.

## URI формата

Согласно блогу:
```
gpt://<идентификатор_каталога>/qwen3.6-35b-a3b
```

**БЕЗ `/latest`** в конце (в отличие от YandexGPT, где принято писать `/latest`).

В другом источнике (статья про VS Code) формат указан как:
```
gpt://<идентификатор_каталога>/<идентификатор_модели>/latest
```

То есть **оба формата валидны** для разных моделей. У нас сейчас в `.env` стоит `qwen3-14b/latest` — это:
- неправильная версия (нужна Qwen 3.6, не Qwen 3)
- неправильное имя (нужно `qwen3.6-35b-a3b`, не `qwen3-14b`)

## ⚠️ Reasoning mode — критично для нас

Из блога:
> Режим рассуждения: включён по умолчанию, можно отключить в API или UI

**Что это значит для нашего пайплайна:**

| Компонент | Reasoning mode нужен? | Почему |
|---|---|---|
| `MessageAnalyzer` | **❌ ОТКЛЮЧИТЬ** | нужен короткий JSON быстро, размышления = +токены, +latency, +цена |
| Основная LLM (главный ответ) | **🤔 спорно** | размышления могут улучшить качество, но удорожают |
| `ReflectionAgent` | **❌ ОТКЛЮЧИТЬ** | фоновый процесс, не критично для качества рассуждений, экономим |

**Открытый вопрос**: как именно отключать в API? Параметр `reasoning_mode: false`? `enable_thinking: false`? `thinking: false`? Нужен пример.

В Qwen 3 (open-source) это делается через `enable_thinking: false` в `chat_template_kwargs`. Возможно, в Yandex API это передаётся аналогично или через какое-то поле в payload.

**Это вопрос #1 для следующего раунда инфы.**
