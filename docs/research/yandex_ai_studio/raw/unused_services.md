# Yandex Cloud сервисы, которые мы НЕ используем (raw, для справки)

> Сохранено для полноты картины. Если в будущем понадобится — здесь сводка.
> Сохранено: Сессия 22, Май 2026

## Yandex Search API — НЕ используем

Поиск Яндекса через API. Может пригодиться **после MVP** для:
- мониторинг упоминаний продукта
- агенты знаний (поиск академических статей, дополнение к PubMed для ResearcherAgent)

**Не подходит для MVP**, потому что:
- продукт не должен «гуглить» во время кризиса (нерелевантные ответы)
- генеративный режим использует встроенный YandexGPT (с цензурой)

### Квоты (для справки)

- 10 000 синхронных запросов/час
- 10 RPS синхронных
- **Только 1 RPS** на запросы с генеративным ответом
- 1 000 запросов в час с генеративным ответом
- Изображения — отдельные квоты

### Лимиты

- Не более 250 результатов на запрос
- Длина запроса до 400 символов / 40 слов
- Хранение результатов отложенных запросов: 12 часов
- Минимальное время обработки в отложенном режиме: 5 минут

### Роли

- `search-api.webSearch.user` — единственная актуальная сервисная роль.
- `search-api.executor` / `auditor` / `viewer` / `editor` / `admin` — **устарели**, не использовать.

## Yandex Translate — НЕ используем

Перевод через API. Не нужно: продукт **только русскоязычный** (по позиционированию).

### Квоты

- 20 RPS на один метод
- 1 млн символов в час

### Лимиты

- Глоссарий: до 50 пар, до 20 000 символов Unicode (по 10 000 на каждую сторону)

## Yandex Vision OCR — НЕ используем

Распознавание текста на изображениях/PDF. Не нужно для MVP: пользователи не загружают изображения.

**Может пригодиться после MVP**, если добавим:
- загрузку фото лекарств (для контекста)
- скриншотов соцсетей (для контекста утраты в Цифровом Двойнике)

### Что есть

- Модели: `page` (по умолчанию), `page-column-sort`, `handwritten`, `table`, `markdown`, `math-markdown`
- Шаблонные документы: `passport`, `driver-license-front/back`, `vehicle-registration-front/back`, `license-plates`
- Языки: автоопределение (модели `handwritten` и `table` — только русский+английский)
- Форматы файлов: JPEG, PNG, PDF
- Размер: до 10 МБ, до 20 мегапикселей
- Режимы: синхронный и асинхронный
- Возвращает: координаты bounding boxes для слов, строк, блоков, изображений, таблиц + полный текст + Markdown

## API Gateway — пока не используем, но скоро понадобится

API Gateway = HTTPS-прокси перед нашим backend.

**Понадобится в Блоке E (deploy)**, если будем выставлять FastAPI наружу через Yandex API Gateway вместо Nginx. Альтернатива в нашем плане — Nginx на VPS Timeweb Cloud.

### Тарификация (для справки)

- Первые 100 000 запросов / месяц — бесплатно
- Далее: **142,3 ₽ за 1 млн запросов**
- Исходящий трафик: первые 100 ГБ/мес — бесплатно, далее 1,42 ₽/ГБ

### Пример

10 млн запросов = (10 000 000 - 100 000) / 1 000 000 × 142,3 ₽ = **1 408,77 ₽/мес**

При наших объёмах (закрытая бета 20-50 человек × 50 сообщений/день × 30 дней = ~75 000 запросов) — попадаем в бесплатный лимит. Поэтому API Gateway — **не приоритет** для MVP, можно оставить Nginx на VPS.

## Object/Operation pattern — концептуальная заметка

Yandex Cloud использует паттерн **Operation object** для всех долгих операций. Это **не тарифицируется**, но важно знать:

- Каждое изменение состояния ресурса возвращает объект `Operation` с `id`, `status`, `created_at`.
- По `Operation.id` можно опрашивать статус.
- В нашем коде это влияет на работу с **дообучением** (если когда-то будем) и **выделенными инстансами** (если перейдём на dedicated).

Для синхронного режима генерации текста через `/v1/chat/completions` — Operation pattern не используется, ответ возвращается сразу.
