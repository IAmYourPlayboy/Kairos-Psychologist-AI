# Yandex SpeechKit — Распознавание речи (raw, для будущих фаз)

> Источник: документация Yandex SpeechKit
> Сохранено: Сессия 22, Май 2026

> ⚠️ Не используем сейчас. Будет нужно в Фазе 7 (Цифровой двойник, голосовое общение)
> и потенциально в Фазе G4 (ElevenLabs TTS — но там голос _бота_, а STT — голос пользователя).

---

## Что это

Speech-to-text — преобразование речи в текст.

Эндпоинт сервиса: `stt.api.cloud.yandex.net:443`

---

## Способы распознавания (3 варианта)

| Тип | Сценарии для нас | Подходит для |
|---|---|---|
| **Потоковое** | Голосовой режим Кайроса в реальном времени | Виртуальные ассистенты, телефонные ассистенты |
| **Синхронное** | Распознавание короткого голосового сообщения, отправленного пользователем | Короткие голосовые в мессенджерах |
| **Асинхронное** | Транскрипция длинной аудиозаписи | Транскрибация длинных записей, контроль скриптов колл-центра |

---

## Сравнение режимов

| Параметр | Потоковое | Синхронное | Асинхронное |
|---|---|---|---|
| Сценарии | Виртуальные ассистенты | Короткие сообщения | Транскрибация записей |
| Входные данные | Голос в реальном времени | Предзаписанные одноканальные короткие | Многоканальные длинные |
| Принцип работы | Обмен сообщениями в одном соединении | Запрос → быстрый ответ | Запрос → отложенный ответ |
| API | gRPC v2, gRPC v3 | REST v1 | REST v2, REST v3, gRPC v3 |
| **Макс. длительность** | **5 минут** | **30 секунд** | **4 часа** |
| Макс. размер файла | 10 МБ | 1 МБ | 1 ГБ |
| Каналов | 1 | 1 | 2 |

## Процесс распознавания

Происходит в три этапа:

1. **Акустическая модель** — какой набор низкоуровневых признаков соответствует звуковому сигналу
2. **Языковая модель** — генерирует текст по словам на основе выхода акустической модели
3. **Постобработка** — расстановка пунктуации, преобразование числительных в цифры

---

## Точность

Зависит от:
- модели распознавания (можно дообучать, см. «Расширение модели распознавания речи»)
- качества исходного звука
- качества кодирования аудио
- разборчивости и темпа речи
- сложности фраз и их длины

## Что нам понадобится в будущем

### Фаза 7 (Цифровой двойник)

При создании двойника:
- Родственники присылают аудиозаписи близкого человека (для клонирования голоса через ElevenLabs)
- **Опционально**: транскрипция этих записей через **асинхронное распознавание** — чтобы автоматически собрать «характерные фразы» близкого

### Голосовой режим терапии (G4 + Фаза 7)

При общении с двойником голосом:
- Пользователь говорит → **потоковое STT** → текст
- Кайрос/Двойник отвечает → ElevenLabs TTS → аудио
- Цикл повторяется

### Не пригодится в MVP

В тексте позиционирования Кайроса (CLAUDE.md):
> Печатание текста заземляет (активное действие). Голос — пассивное восприятие. **Поэтому в кризисе текст > голос**.

То есть для **ППП-режима** голос вообще не нужен. Только для двойника.
