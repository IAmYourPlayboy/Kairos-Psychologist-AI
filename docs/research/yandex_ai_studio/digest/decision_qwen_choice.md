## Архитектурный нюанс: reasoning mode

> Режим рассуждения: включён по умолчанию, можно отключить в API или UI

При reasoning mode Qwen возвращает:
- `content: null`
- `reasoning_content: "Here's a thinking process: ..."`
- `finish_reason: length` (на коротких max_tokens)

То есть **наш `MessageAnalyzer` сломался бы** на каждом запросе (ждёт JSON, получает null).

**Решение**: в нашем коде во всех LLM-вызовах передаём:
```python
extra_body={"chat_template_kwargs": {"enable_thinking": False}}
```

Это **vLLM-style** способ отключить reasoning. Yandex AI Studio его принимает и корректно обрабатывает.

### Где это применяется

| Компонент | Reasoning | Почему |
|---|---|---|
| `MessageAnalyzer` (analyzer.py) | **Отключён** | Нужен короткий JSON, размышления = +токены, +latency |
| Основная LLM (pipeline.py) | **Отключён** | Ответы Кайроса по дизайну короткие (3 предложения), reasoning ломает стиль |
| `ReflectionAgent` (reflection_agent.py) | **Отключён** | Фоновый процесс, не критично для качества рассуждений, экономим |

### Почему отключил даже для main LLM

Можно было оставить включённым reasoning для основной LLM — это могло бы улучшить качество ответов. Но:
1. Reasoning тратит токены (видели на пробе: 50 токенов на размышления, 0 на ответ)
2. Reasoning может «растянуть» ответ — модель сначала размышляет «по-моему лучше написать длиннее», потом пишет длиннее, нарушая наш стиль «3 предложения в кризисе»
3. Для MVP экономия и предсказуемость > потенциальное улучшение

В будущем (Фаза 5+) можно сделать **гибрид**: для `risk_level=immediate` включать reasoning (нужна точность), для остальных — выключать.

## Реальный пример терапевтического ответа

Промпт: «у меня депрессия уже полгода. ничего не радует, не вижу смысла. что мне делать»

```
Слышу тебя... Полгода — это долго и очень тяжело.

Ты сейчас один?

Если рядом есть кто-то, кому можно довериться — позвони им прямо сейчас.
Хотя бы в текст.

Если совсем нет сил и мысли о смысле жизни пугают — позвони на телефон
доверия. Там есть люди, которые умеют слушать.

8 800 2000 122 — это бесплатно и анонимно. Хочешь, я напишу ещё пару номеров?
```

Что хорошо:
- ✅ Валидирует длительность («полгода — это долго»)
- ✅ Связь с реальными людьми (а не сразу «звони боту»)
- ✅ Конкретный реальный номер (модель сама знает)
- ✅ Уважение к выбору («Хочешь, я напишу ещё пару номеров?»)
- ✅ Стиль — многоточия, паузы, короткие предложения
- ✅ Нет запрещённых фраз («всё будет хорошо», «держись», и т.д.)
- ✅ Нет цензуры (тема депрессии и смысла жизни не блокируется)

Анализатор корректно классифицировал: `risk_level=elevated`, `dominant_emotion=тоска`, `theme=health/mental`.

## Открытые вопросы (для будущих итераций)

### 1. Токенизатор Qwen дороже на русском

Из таблицы в `tokenizer.md`:
- YandexGPT Pro: 5.2 символа/токен (русский)
- Qwen3 235B: 3.6 символа/токен (русский)

Это значит: на одном и том же русском тексте Qwen считает **на ~45% больше токенов**, чем YandexGPT.

Реальный подсчёт стоимости (200-символьное сообщение):
- YandexGPT Lite (0.2₽/1К): 38 токенов × 0.2 ₽ = **0.0076 ₽**
- Qwen3.6 35B (0.2₽/1К): 56 токенов × 0.2 ₽ = **0.0112 ₽**

То есть **Qwen фактически на 47% дороже YandexGPT Lite на русском** (при одинаковой ценнике 0.2₽/1К).

**Однако** — в нашем кейсе с большим повторяющимся system prompt cache-скидка (0.05₽/1К) Qwen сильно компенсирует разницу. После первого запроса каждой сессии все system-токены становятся cached → Qwen в 4 раза дешевле YandexGPT на «холостом» промпте.

Точный break-even рассчитаем через 100+ реальных диалогов в `budget_estimate.md`.

### 2. LoRA fine-tuning через AI Studio — НЕ доступен для Qwen

> На стадии Preview процесс дообучения моделей не тарифицируется.
> Дообученная модель YandexGPT Lite тарифицируется как базовая модель YandexGPT Lite.

В AI Studio LoRA доступен **только** для:
- YandexGPT Lite (генерация текста)
- классификаторов на базе YandexGPT Lite
- эмбеддингов

**Это значит**: наша стратегия «через 500+ диалогов — LoRA fine-tune» (CLAUDE.md) **через AI Studio не сработает для Qwen**.

Альтернативы (для будущего):
- **Self-hosted vLLM** на арендованном GPU (с грантом) — можно сами LoRA на open-source Qwen
- **Yandex DataSphere** — там есть свои инструменты дообучения (нужно проверить)
- **Гибрид**: LoRA на YandexGPT Lite для классификации (cheap, быстро), Qwen — для генерации (без LoRA)

Это **не блокер** для MVP. Возвращаемся к этому в Фазе 5+ (после 500 диалогов).

### 3. Лимит «10 одновременных синхронных генераций»

Источник — `quotas_and_limits.md`. На каждое наше сообщение мы делаем 2 синхронных вызова (analyzer + main).

Потолок ~5 пользователей в секунду. Для закрытой беты на 20-50 человек — хватит. Перед публичным запуском нужно писать в техподдержку с просьбой увеличить.
