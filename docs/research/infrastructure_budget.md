## Qwen: какую модель выбрать и сколько VRAM нужно

Qwen 2.5 72B в полной точности (FP16) требует **~144 ГБ VRAM** — это два A100 80GB. С AWQ 4-bit квантизацией вес модели сжимается до **~36–38 ГБ**, а с учётом KV-кэша нужно **40–47 ГБ** — помещается на один A100 80GB, но не на RTX 4090 (24 ГБ). При бюджете 10 000₽/мес аренда A100 стоит от 60 000₽ — это вне досягаемости.

Однако появился критически важный фактор: **Qwen 3 14B по качеству сопоставим с Qwen 2.5 72B** при пятикратно меньшем размере. Это кардинально меняет расклад. Qwen 2.5/3 14B с 4-bit квантизацией занимает всего **8–10 ГБ VRAM** и комфортно работает на RTX 3090/4090 (24 ГБ) или даже Tesla T4 (16 ГБ). Скорость инференса — **25–35 токенов/сек** на RTX 4090, что достаточно для интерактивного чат-бота.

Качество Qwen на русском языке — одно из лучших среди open-source моделей. Русский входит в **29+ официально поддерживаемых языков**, модель обучена на 18 триллионах токенов мультиязычных данных. Qwen 2.5 72B обгоняет Llama 3.1 405B (модель в 5 раз крупнее) на ряде бенчмарков. Для терапевтического бота **14B — минимально рекомендуемый размер**: 7B-версия справится с базовым диалогом, но для нюансированной терапевтической беседы будет недостаточно глубокой.

Важное исследование (EMNLP 2025) показало, что 4-bit квантизация **деградирует качество на нерусском/неанглийском тексте до 5 раз сильнее**, чем на английском. Однако **Qwen 2.5 72B оказался исключительно устойчив** — при BNB-nf4 показал Δ-accuracy ≈ +0.6% (практически без потерь), тогда как Llama 3.1 70B потерял 32%. Для 14B модели рекомендуется **Q5_K_M** квантизация вместо Q4 — качество выше при незначительном росте VRAM (10–12 ГБ вместо 8–9 ГБ).

### Движки инференса: что выбрать

- **Ollama** — лучший выбор для старта и прототипирования. Простая установка, OpenAI-совместимый API из коробки, поддержка GGUF-квантизации. Достаточен для 1–5 одновременных пользователей
- **vLLM** — стандарт для продакшена с несколькими пользователями. PagedAttention оптимизирует память, continuous batching обрабатывает параллельные запросы. На 16 одновременных пользователях **на 23% быстрее** llama.cpp
- **llama.cpp** — максимальный контроль, поддержка CPU-offloading. Быстрее vLLM на single-user сценариях (на 4,7%), но хуже масштабируется

## Inference-as-a-service: главный секрет бюджетного запуска

Вместо аренды GPU-сервера за 10 000₽/мес можно использовать API-доступ к моделям — и уложиться в **1 000–3 000₽/мес** при умеренной нагрузке. Это главная находка исследования.

**Yandex Cloud AI Studio** предоставляет Qwen 3 модели через API с российским хранением данных:

| Модель | Синхронный режим (за 1K токенов) | Пакетный режим |
|--------|----------------------------------|----------------|
| Qwen3-8B | ~0,20₽ | **0,10₽** |
| Qwen3-14B | ~0,40₽ | **0,20₽** |
| Qwen3-32B | ~0,80₽ | **0,41₽** |
| Qwen3-235B (MoE) | ~1,00₽ | **6,10₽** |

При средней длине диалога ~500 токенов и 1 000 диалогов в месяц расход составит всего **100–400₽** на модель Qwen3-14B. Даже при 10 000 диалогов — **1 000–4 000₽/мес**. Плюс стартовый грант **4 000₽ на 60 дней** для новых аккаунтов.

**Cloud.ru (бывший SberCloud)** открыл **бесплатный доступ** к каталогу Evolution Foundation Models, включая Qwen3-235B, GPT-OSS-120B и GigaChat Lightning. API совместим с OpenAI — можно подключить как drop-in замену. При регистрации дают **4 000–20 000 бонусов** (физлица/юрлица). Данные хранятся в России — полное соответствие ФЗ-152.

**GigaChat API** (Сбер) даёт **1 000 000 бесплатных токенов на 12 месяцев** для физлиц (900K Lite + 50K Pro + 50K Max). Платные пакеты стартуют от **1 300₽ за 20 млн токенов** (GigaChat 2 Lite). Однако для терапевтического бота GigaChat **категорически не подходит** — агрессивная цензура блокирует обсуждение эмоционально сложных тем (депрессия, тревога, суицидальные мысли), что прямо противоречит целям терапевтического приложения. YandexGPT имеет аналогичные, хотя менее агрессивные ограничения.

**Оптимальная стратегия для терапевтического бота:** использовать **open-source Qwen через Yandex Cloud AI Studio или Cloud.ru** — вы контролируете system prompt, нет встроенной цензуры уровня GigaChat, данные в России.

## VPS для бекенда: 1 000₽/мес — достаточно

Docker Compose стек (Nginx + Next.js + FastAPI + PostgreSQL + Redis) потребляет **~1,4–2,8 ГБ RAM**. VPS с 2 vCPU / 4 ГБ RAM справляется с нагрузкой до ~100 одновременных пользователей.

**Timeweb Cloud** — лидер по цене/качеству: 2 vCPU / 4 ГБ / 50 ГБ NVMe за **1 000₽/мес**, маркетплейс Docker-образов, серверы в Москве/СПб/Новосибирске, подтверждённое соответствие ФЗ-152. Альтернативы: **Beget** от 840₽/мес (30 дней бесплатного тестирования), **REG.RU** от 1 200₽/мес с удобной экосистемой домен+хостинг. При росте нагрузки масштабирование у Timeweb до 4 vCPU / 8 ГБ / 80 ГБ — ~2 200₽/мес.

**Домен .ru** обойдётся в **129₽/год** по акции REG.RU (продление — 999₽/год, или перенести к Hostiman за 400₽/год). **SSL** — бесплатно через Let's Encrypt с автопродлением через certbot. **CDN** — Cloudflare Free (DNS + прокси + DDoS-защита) — бесплатно.

### ФЗ-152 и выбор архитектуры

По ФЗ-152 первичные запись, хранение и обработка персональных данных граждан РФ допускается **только на серверах в РФ**. Штрафы — до **6 млн₽** за нарушение локализации. Практические выводы:

- **Cloudflare как CDN/прокси перед российским VPS** — допустимо, данные хранятся на VPS в РФ, Cloudflare только кеширует статику
- **Vercel/Cloudflare Pages для фронтенда** — формально допустимо, если фронтенд не хранит/обрабатывает ПДн, а все данные идут напрямую на API в РФ. Но SSR на Vercel, получающий ПДн из API — серая зона
- **Безопасный подход:** размещать и фронтенд, и бэкенд на одном VPS в РФ через Docker Compose, а Vercel использовать только как CDN для статики

## Aniemore: распознавание эмоций на CPU за 10–50 мс

**Aniemore** — единственная специализированная библиотека для распознавания эмоций в русском тексте и речи. Проект жив: последнее обновление — **июнь 2025**, флагманская модель скачивается **~4 232 раза в месяц** на HuggingFace. MIT-лицензия.

Ключевая модель — `rubert-tiny2-russian-emotion-detection` (29,2 млн параметров): **85% точности** на 7 классах эмоций (радость, грусть, злость, страх, отвращение, энтузиазм, нейтрально). Для голоса лучшая модель WavLM достигает **82%**, мультимодальная (голос+текст) — **83%**.

Все текстовые модели Aniemore **работают на CPU** без GPU. Требования минимальны: **2 ядра, 4 ГБ RAM, ~120 МБ на модель**. Инференс одного текста занимает **10–50 мс** на 2 vCPU — вполне достаточно для real-time обработки в чат-боте. Модель помещается на один VPS вместе со всем остальным стеком.

Альтернатива — `cointegrated/rubert-tiny2-cedr-emotion-detection` от известного российского NLP-исследователя David Dale: **AUC 0,8956**, F1 micro 0,928, 6 классов эмоций. Для более детальной классификации (28 эмоций) есть `seara/rubert-tiny2-ru-go-emotions`, но обучена на машинно-переведённом датасете — качество на нативном русском может быть ниже. Библиотеку **Dostoevsky** использовать не стоит — архивирована в октябре 2023.

Для терапевтического бота рекомендуется включить Aniemore в FastAPI-бекенд: анализировать эмоции пользователя в каждом сообщении и передавать результат как контекст в system prompt LLM — это позволит модели адаптировать тон ответа.

### Фазы разработки

**Месяц 1 — Прототип (бюджет: 0₽).** Локальная разработка: Ollama + Qwen3-14B на ПК (если есть GPU 12+ ГБ) или Groq/Together AI бесплатные тиры для тестирования промптов. GigaChat Freemium (1M токенов) — для сравнения качества. Cloud.ru бесплатный каталог моделей — для тестирования Qwen3 на российской инфраструктуре. Фронтенд Next.js + бекенд FastAPI разрабатывать локально.

**Месяц 2 — Alpha на VPS (бюджет: ~2 000₽).** Развернуть Docker Compose на Timeweb Cloud. Подключить Yandex Cloud AI Studio API для Qwen3-14B. Интегрировать Aniemore для анализа эмоций. Настроить Let's Encrypt + Nginx. Начать закрытое тестирование.

**Месяц 3 — MVP (бюджет: ~3 000–5 000₽).** Подать заявку на Yandex Cloud Boost Start (50K₽ грант). Оптимизировать промпты для терапевтического сценария. Масштабировать VPS до 4 vCPU / 8 ГБ при необходимости. Открытый запуск.

**Месяц 4 — Рост (бюджет: покрывается грантом).** Если грант одобрен — перейти на Qwen3-32B для лучшего качества. Рассмотреть fine-tuning через LoRA/QLoRA на терапевтических диалогах. Подать заявку на Yandex Cloud Boost AI (до 1M₽).

## Заключение

Главный вывод: **собственный GPU-сервер не нужен для MVP**. Появление inference-as-a-service у российских провайдеров (Yandex Cloud AI Studio за 0,10–0,41₽/1K токенов, Cloud.ru с бесплатным доступом к Qwen3) радикально снижает порог входа. Qwen 3 14B обеспечивает качество уровня Qwen 2.5 72B при стоимости инференса в **20–40 копеек за тысячу токенов**. Для терапевтического бота критически важно использовать именно open-source модели через API, а не GigaChat/YandexGPT — встроенная цензура российских проприетарных моделей заблокирует обсуждение эмоционально сложных тем, что обессмыслит приложение. Стартап-гранты Yandex Cloud (50K–1M₽) могут полностью покрыть инфраструктуру на год — подавать заявку стоит сразу после запуска MVP.
