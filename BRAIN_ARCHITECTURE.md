## АРХИТЕКТУРА

```
НАУЧНЫЕ PDF (статьи, протоколы, исследования)
    ↓
ОБРАБОТКА ОДИН РАЗ (вручную извлекаются ключевые знания)
    ↓
МОЗГ КАЙРОСА (статичные файлы в backend/app/core/knowledge/)
    ├── crisis_situations.py — 455+ кризисных ситуаций с keywords
    ├── who_pfa.py — протокол ВОЗ (Look, Listen, Link + заземление)
    ├── six_cs.py — протокол SIX C's Моше Фарчи (мобилизация)
    ├── cbt_techniques.py — когнитивно-поведенческая терапия
    ├── dbt_skills.py — диалектическая поведенческая терапия
    ├── act_processes.py — терапия принятия и ответственности
    └── sfbt_mi.py — краткосрочная терапия + мотивационное консультирование
    
    ⚠️ ЮЗЕР НЕ ВИДИТ ЭТОТ СЛОЙ
    
МОДУЛИ (руки Кайроса, взаимодействуют с юзером)
    ├── therapy_router.py — маршрутизация терапевтических техник
    ├── crisis/detector.py — детекция кризиса через keywords
    ├── prompts/builder.py — сборка промптов из знаний мозга
    └── [эти модули ИМПОРТИРУЮТ знания из мозга при запуске]

DATA FLYWHEEL (PostgreSQL)
    └── Сбор данных от реальных юзеров → через 500+ диалогов → LoRA fine-tuning
```

## КЛЮЧЕВЫЕ ОТЛИЧИЯ ОТ RAG

| RAG (векторный поиск) | Мозг Кайроса (статичные модули) |
|------------------------|----------------------------------|
| Каждый запрос → поиск в Qdrant | Знания загружены ОДИН РАЗ при импорте |
| Лаг ~100-200мс на каждый запрос | Лаг 0мс (знания уже в памяти Python) |
| Юзер видит научные ссылки | Юзер НЕ видит мозг, только естественный ответ |
| Ответ в научном стиле | Ответ в человеческом стиле (но на научной базе) |
| Нужен Qdrant/векторная БД | Нужен только Python (импорт модулей) |

## СТЕНКА МЕЖДУ МОЗГОМ И ЮЗЕРОМ

Юзер **НЕ получает** ответы вида:
> "Согласно исследованию Van der Kolk (2014), ПТСР связан с гиперактивацией амигдалы..."

Юзер **получает** ответы вида:
> "Слышу, что сейчас очень тяжело. Давай попробуем вернуться в настоящий момент. Посмотри вокруг и назови мне 5 вещей, которые ты видишь."

**Почему это работает:**
Кайрос УЖЕ ЗНАЕТ из WHO PFA Guide (2011), что техника 5-4-3-2-1 эффективна при панике и диссоциации. Это знание встроено в модуль `who_pfa.py`, который импортируется в `prompts/builder.py`, который формирует system prompt для LLM.

## КАК ДОБАВЛЯТЬ НОВЫЕ ЗНАНИЯ

### Шаг 1: Получить научный источник (PDF)
Например: WHO "Psychological First Aid Guide" (2011)

### Шаг 2: Извлечь ключевые знания (вручную)
Читаешь статью и выделяешь:
- Протоколы (последовательность действий)
- Техники (конкретные приёмы)
- Примеры фраз (что говорить)
- Противопоказания (что НЕ говорить)
- Научное обоснование (почему это работает)

### Шаг 3: Обновить соответствующий модуль в `knowledge/`
Например, из WHO PFA Guide извлекаешь:
- Look, Listen, Link → добавляешь в `who_pfa.py`
- Техники заземления → добавляешь в `who_pfa.py`
- Запрещённые фразы → добавляешь в `prompts/forbidden_phrases.py`

### Шаг 4: Модули автоматически используют обновлённые знания
После обновления `who_pfa.py`:
- `therapy_router.py` импортирует обновлённые техники
- `prompts/builder.py` использует их для формирования промптов
- LLM получает обновлённый system prompt
- Юзер получает более точные ответы

**Никакого лага, никакого RAG, никакого поиска в реальном времени.**

## DATA FLYWHEEL (отдельная система)

**Мозг Кайроса** и **Data Flywheel** — это **две разные системы**:

### Мозг Кайроса (статичные знания):
- Источник: научные статьи, протоколы, исследования
- Обновление: вручную, при добавлении новых источников
- Назначение: научная база для правильных ответов
- Хранение: Python модули в `backend/app/core/knowledge/`

### Data Flywheel (динамические данные):
- Источник: реальные диалоги с пользователями
- Обновление: автоматически, каждый диалог
- Назначение: находить паттерны, что работает лучше
- Хранение: PostgreSQL → через 500+ диалогов → LoRA fine-tuning

**Пример:**
- Мозг знает: "Техника 5-4-3-2-1 эффективна при панике" (из WHO PFA Guide)
- Data Flywheel показывает: "В российском контексте техника работает лучше если начинать с 3 предметов, а не 5" (из реальных диалогов)
- Результат: обновляем `who_pfa.py` на основе инсайтов из Data Flywheel

## ВАЖНО ДЛЯ AI-АССИСТЕНТОВ

Когда пользователь говорит "добавь эту статью в мозг Кайроса":

1. **НЕ предлагай RAG** (векторный поиск, Qdrant, embeddings)
2. **НЕ предлагай** добавлять научные ссылки в ответы юзеру
3. **ПРЕДЛОЖИ** извлечь ключевые знания и обновить модули в `knowledge/`
4. **ПОМНИ** что юзер не видит мозг — только естественные ответы

Когда пользователь спрашивает "как работает мозг Кайроса":

1. **Объясни** что это статичные Python модули, а не векторная БД
2. **Покажи** что знания загружаются один раз при импорте (0мс лага)
3. **Подчеркни** стенку между мозгом и юзером (юзер не видит научные ссылки)
