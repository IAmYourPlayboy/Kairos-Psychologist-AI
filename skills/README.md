## Что такое скиллы

Скиллы — это специализированные знания и паттерны для разработки AI-Психолог. Каждый скилл содержит:
- Конкретные инструкции и правила

## Обязательные скиллы (применять ВСЕГДА)

### 1. behavioral-guidelines.md
**Когда**: При написании ЛЮБОГО кода
**Что**: Принципы качественного кода от Андрея Карпатого
- Think Before Coding — явно формулируй предположения
- Simplicity First — минимум кода, никаких спекуляций
- Surgical Changes — трогай только то, что нужно
- Goal-Driven Execution — определи критерии успеха

**Как применять**:
```
Перед написанием кода:
1. Сформулируй предположения явно
2. Спроси себя: "Можно ли проще?"
3. Определи критерий успеха (как проверить, что работает?)
4. Пиши минимум кода для решения задачи
```

### 2. ai-psychologist-core.md
**Когда**: При любой работе над проектом AI-Психолог
**Что**: Мастер-контекст проекта
- Два режима (ППП — базовый чат Кайроса + Цифровой двойник)
- Нейробиологическое обоснование (amygdala hijack)
- Экосистема всех скиллов
- Технический стек

**Как применять**: Читай в начале каждой сессии, чтобы понять контекст проекта.

## Специализированные скиллы (применять по контексту)

### Терапевтические промпты

**Скилл**: `therapeutic-prompts.md`

**Когда использовать**:
- Пишешь системные промпты для бота
- Создаёшь диалоги SIX C's или ВОЗ
- Работаешь с кризисными сценариями
- Проектируешь ответы бота

**Ключевые правила**:
```
ЗАПРЕЩЁННЫЕ ФРАЗЫ (НИКОГДА не использовать):
- "Я понимаю, что ты чувствуешь" (бот не чувствует)
- "Всё будет хорошо" (обесценивание)
- "Тебе нужно успокоиться" (обесценивание)
- "Держись" (пустое)
- "Бывает и хуже" (минимизация)
- "Ты сильный/сильная" (навязывание ожидания)
- "Что ты чувствуешь?" в остром кризисе (усиливает затопление)

ПРАВИЛА:
- В кризисе: максимум 3 предложения на сообщение
- Мат/сленг — это маркеры дистресса, не агрессия
- Бот не поправляет речь, не выражает неодобрение
- Обращение на "ты"
```

**Структура промпта SIX C's**:
```
Challenge → Control → Commitment → Continuity → Calmness

Challenge: "Найди 3 предмета жёлтого цвета вокруг себя"
Control: "Мы можем сделать дыхательное упражнение или упражнение на мышцы. Что выберешь?"
Commitment: "Обещай, что как только мы закончим, ты выпьешь стакан воды"
Continuity: "Вспомни, что ты делал за час до того, как это началось?"
Calmness: "Почувствуй, как твои пятки давят на пол. Считай до 10"
```

### NLP и эмоции

**Скилл**: `nlp-emotion-analysis.md`

**Когда использовать**:
- Разрабатываешь NLP-пайплайн
- Работаешь с Aniemore
- Создаёшь детекцию дистресса
- Проектируешь матрицу тема×интенсивность

**Архитектура**:
```python
# Два параллельных классификатора

# Layer 1: Distress Markers (0.0 → 1.0)
# КАК человек говорит = термометр дистресса
markers = {
    "absolutist": ["никогда", "всегда", "никто", "ничего"],
    "hopelessness": ["бессмысленно", "невозможно", "конец"],
    "slang_distress": ["пздц", "хз", "мне п***ц", "капец"],
    "caps_lock": True,  # >50% заглавных
    "message_shortening": True,  # прогрессивное сокращение
}

# Layer 2: Thematic Categories
# О ЧЁМ человек говорит = диагноз
themes = [
    "утрата",      # → ACT + предложение двойника
    "вина",        # → CBT
    "одиночество", # → связь
    "безысходность", # → КРАСНАЯ ЗОНА
    "страх",       # → SIX C's
    "гнев",        # → DBT
    "тревога",     # → дыхание + CBT
]

# Матрица пересечений
if theme == "безысходность" and distress > 0.7:
    action = "CRISIS_ROUTING"  # Красная зона
elif theme == "утрата" and distress < 0.3:
    action = "ACT_GENTLE"  # Нормальное горе
elif distress > 0.85:
    action = "VALIDATE_THEN_CHECK"  # Любая тема + высокий дистресс
```

**Толерантность к сленгу**:
```python
# Модель ДОЛЖНА распознавать как маркеры дистресса:
distress_markers = [
    "пздц", "капец", "мне п***ц",  # высокий дистресс
    "хз что делать",                # беспомощность
    "всё, я больше не могу",        # без пунктуации = тот же вес
    "CAPS LOCK + rapid messages",   # паника
    "всё хорошо 🙃",                # сарказм = противоречие
]
```

- Проектируешь эскалацию

**Уровни детекции**:
```python
# 3 уровня кризиса

IMMEDIATE = {
    "keywords": ["хочу умереть", "покончить с собой", "суицид"],
    "action": "SHOW_CONTACTS_IMMEDIATELY",  # Без валидации
}

HIGH = {
    "keywords": ["безысходность", "нет выхода", "всё бессмысленно"],
    "action": "VALIDATE_THEN_ESCALATE",  # "Тебе сейчас тяжело. Я прав?"

ELEVATED = {
    "markers": ["дистресс", "паника", "не могу"],
    "action": "ELEVATED_MONITORING",  # Повышенный мониторинг

```

### Цифровой двойник

**Скилл**: `grief-module.md`

**Когда использовать**:
- Работаешь с модулем двойника
- Проектируешь механику угасания
- Создаёшь анкету для родственников

**Lifecycle двойника**:
```
1. Профиль (анкета + аудио) → системный промпт + ElevenLabs PVC
2. Активный период (пользователь общается с двойником)
3. Адаптивное угасание (3-12 мес, привязано к distress score)
4. Финальное прощание от первого лица двойника
5. Бесшовный переход в терапевтический режим
```

**Эвристика угасания**:
```python
fade_out_config = {
    "base_duration": 7,  # месяцев
    "min_duration": 3,   # минимум перед началом угасания
    "max_duration": 12,  # максимум → мягкое завершение
    
    "start_condition": {
        "avg_distress_2weeks": "<0.3",
        "min_time_passed": "3 months",
    },
    
    "pause_condition": {
        "distress_spike": ">0.6",
        "pause_duration": "2 weeks",
    },
    
    "voice_phases": {
        "active": {"stability": 0.5},
        "early_fade": {"stability": 0.6},
        "mid_fade": {"stability": 0.7},
        "late_fade": {"stability": 0.9},  # голос спокойнее, отдалённее
    },
}
```

**Поведение двойника в кризисе**:
```
Этап 1 — Заземление в характере (elevated/high):
"Поставь ноги на пол. Почувствуй, как они стоят. Я здесь."
(Протоколы ВОЗ, но языком живого человека)

Этап 2 — Мягкая передача (immediate или этап 1 не помог):
"Мне кажется, тебе нужна помощь, которую я дать не могу. 
Вот номера людей, которые помогут."

ЗАПРЕТ: НИКОГДА не манипулировать виной ("Мама просит тебя остаться")
```

### Отладка

**Скилл**: `systematic-debugging.skill`

**Когда использовать**:
- Отлаживаешь баги
- ML-модель работает неправильно
- Непонятное поведение системы

**4 фазы отладки**:
```
1. Investigation (Расследование)
   - Воспроизведи баг
   - Собери факты
   - НЕ предполагай причину

2. Diagnosis (Диагноз)
   - Сформулируй гипотезу
   - Проверь гипотезу
   - Найди root cause

3. Treatment (Лечение)
   - Исправь причину, не симптом
   - Минимальное изменение
   - НЕ обходи проблему

4. Verification (Верификация)
   - Проверь, что баг исправлен
   - Проверь, что ничего не сломалось
   - Добавь тест, чтобы баг не вернулся
```

**Главное правило**: "Никаких фиксов без причины" — сначала найди root cause, потом исправляй.

## Связи между скиллами

Скиллы работают как единая экосистема. Пример цепочки:

```
Пользователь: "мне плохо, хз что делать, всё бессмысленно"
    ↓
nlp-emotion-analysis:
  - Layer 1 (маркеры): "хз" + "бессмысленно" → distress_score = 0.75
  - Layer 2 (тема): "безысходность"
  - Матрица: безысходность × 0.75 = КРАСНАЯ ЗОНА
    ↓
crisis-routing:
  - Уровень: HIGH
  - Действие: валидация ("Тебе сейчас тяжело. Я прав?")
    ↓
therapeutic-prompts:
  - Ветка Б (стабилизация)
  - Протокол ВОЗ: заземление + дыхание
  - Фразы из Calmness
    ↓
crossplatform-frontend:
  - Отображение в чат-интерфейсе
  - SOS-кнопка пульсирует
  - Карточка с контактами
```

## Как работать со скиллами

### При начале новой сессии:

1. **Прочитай обязательные скиллы**:
   ```
   - behavioral-guidelines.md (ВСЕГДА)
   - ai-psychologist-core.md (контекст проекта)
   ```

2. **Определи, какие специализированные скиллы нужны**:
   - Пишешь промпты? → `therapeutic-prompts.md`
   - Работаешь с NLP? → `nlp-emotion-analysis.md`
   - Кризисная детекция? → `crisis-routing.md`
   - Двойник? → `grief-module.md`
   - Баги? → `systematic-debugging.skill`

3. **Применяй правила из скиллов**:
   - Не нарушай запрещённые паттерны
   - Следуй структурам и примерам
   - Используй связи между скиллами

### При написании кода:

```
ПЕРЕД написанием:
1. Behavioral guidelines: "Можно ли проще?"
2. Определи критерий успеха
3. Сформулируй предположения

ВО ВРЕМЯ написания:
1. Минимум кода
2. Никаких спекуляций
3. Surgical changes — трогай только нужное

ПОСЛЕ написания:
1. Проверь критерий успеха
2. Убери лишнее
3. Добавь тест (если нужно)
```
